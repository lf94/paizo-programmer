# Typical usage

* User navigates to the website on their phone
* First view is just blank, with + button near the lower right
* Tapping or dragging the + button anywhere opens a text box
* This text box should take up 1/3rd of the screen
* The fact or rule goes here.
* When done, the text input should now be displayed as a list item
* When all facts and rules are written, the user swipes up from the bottom
* Another text input appears and a > button near the bottom left
* The new text input should only take up 1/3rd of the screen.
* The question / query goes here.
* Pressing > will send the question, facts and rules to tao-prolog. This does
  not close the question overlay.
* The answers returned from tao-prolog are displayed in an overlay in the
  middle of the screen.
* A stop and more button are beneath this overlay.
* More continues to give more answers from tao-prolog.
* Stop closes the overlay.
